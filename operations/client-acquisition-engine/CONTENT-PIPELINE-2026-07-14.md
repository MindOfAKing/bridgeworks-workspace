## Chrome Publishing Protocol

Emmanuel has confirmed that an agent can publish through his logged-in Chrome browser and upload photo files from the local computer. This is the execution channel when a social connector cannot reach the exact target or when a native upload is required.

Before opening a composer, the approval record must identify:

- Target account and channel.
- Exact final caption or comment.
- Exact local photo or video file paths, if any.
- Destination post URL for a comment or reply.
- Whether the authorization is `prepare preview only` or `publish now`.

For an exact `publish now` authorization, the agent must:

1. Open the approved account in the user's logged-in Chrome session.
2. Recheck the destination, account identity, and current public context.
3. Enter the approved text without adding claims, tags, or links.
4. Upload only the approved local files and verify the preview is complete and correctly cropped.
5. Stop if the account, text, asset, crop, destination, or platform state differs from the approved bundle.
6. Publish only when the bundle still matches the authorization.
7. Capture the live URL or post ID, timestamp, account, and asset path in the content pipeline and weekly scorecard.

Chrome access is a publishing capability, not standing authorization. Draft preparation, browser preview, and public publishing remain distinct states.
