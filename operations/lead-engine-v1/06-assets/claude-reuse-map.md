# Claude Code Reuse Map for BridgeWorks Lead Engine v1

Date: 2026-06-23

Purpose: preserve Emmanuel's useful Claude Code marketing, GEO, and lead-generation assets without letting the old Claude system become the operating system again.

## Operating rule

BridgeWorks Lead Engine v1 is the source of truth for acquisition.

Use old Claude assets as specialist lenses only:

1. Research the market.
2. Pick a test lane.
3. Score prospects.
4. Run a light Lead Leak Review.
5. Send specific outreach only after Emmanuel approves.
6. Prepare for calls.
7. Write proposals only after interest exists.

Do not run the full Claude marketing/GEO machinery on every prospect.

## Recommended research sprint flow

1. **Generate candidate lanes**
   - Use `competitive-research` and `market-competitive` concepts.
   - Output: 8-12 niche/location combinations.

2. **Desk research each lane**
   - Use `market-competitive` for landscape.
   - Use `market-strategy` lightly for ability to pay and urgency.
   - Use `market-content`, `market-conversion`, and `market-technical` only on sampled websites.
   - Output: evidence notes, not a polished report.

3. **Score lanes**
   - Use the current `market-research-framework.md` scorecard.
   - Do not replace it with old Claude scoring.

4. **Build first prospect list**
   - Use current Lead Engine tracker logic.
   - Use `lead-qualifier` only to assign HOT/WARM/COLD and next action.

5. **Run Lead Leak Reviews for selected prospects**
   - Use `client-audit` and market audit agents as checklists.
   - Compress output into the 48-Hour Lead Leak Review format.

6. **Outreach**
   - Use current outreach templates.
   - Use old copy/email skills only to tighten clarity.
   - Emmanuel approves before sending.

7. **Call and proposal**
   - Use `discovery-call-prep` before calls.
   - Use `post-call-followup` after calls.
   - Use `new-proposal` only after scope and price are real.

## What to preserve

- Emmanuel voice rules: no em dashes, short sentences, no AI-slop terms.
- BridgeWorks brand standards for formal documents.
- Lead qualifier logic.
- Discovery call question structure.
- Post-call follow-up discipline.
- Market audit agents as specialist checklists.
- GEO agents as premium diagnostic/delivery tools.
- CEEFM as proof and a past case study only.

## What is stale or dangerous

- Treating CEEFM as active client delivery. Current rule: CEEFM is closed/past case only.
- Session-start/session-end rituals that auto pull, commit, push, or update sheets.
- Full GEO CRM and `~/.geo-prospects` pipeline as the operating layer.
- Apollo/Hunter/Tomba enrichment loops without explicit approval.
- Automated outreach or sending without Emmanuel approval.
- Full marketing audit suite as the default first touch.
- PDF/report generation before there is real interest.
- Old pricing, tiers, or ROI claims that conflict with current BridgeWorks offer ladder.
- Claims about AI visibility growth or guaranteed results that cannot be proven for the prospect.
- Heavy content/social/YouTube engines as acquisition distractions.
- Any draft that assumes client approval, active scope, payment, or public publishing.

## Practical recommendation

Use the Claude assets like a toolbox, not a cockpit.

For the immediate sprint:

- **Use now:** `competitive-research`, `market-competitive`, `lead-qualifier`, `client-audit`, `market-conversion`, `market-content`, `market-technical`, `discovery-call-prep`, `post-call-followup`, `new-proposal`.
- **Use selectively:** `market-strategy`, `market-seo`, `market-report`, `market-proposal`, GEO agents.
- **Park for later:** `geo-prospect`, full GEO proposal/report/PDF flow, content-week, ad-script, social visual generation, YouTube intelligence, automated CRM/dashboard systems.

The lean rule: if an asset does not help find, score, contact, call, or close a real prospect this week, park it.
