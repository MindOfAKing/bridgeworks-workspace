# YouTube Extraction: AI Human Work Queue And Evals

Source: Nick Saraev, `Steal My Actual AI Agent Workflow (2027)`  
URL: https://www.youtube.com/watch?v=8rVQuZlRaqo  
Date extracted: 2026-07-15  
BridgeWorks pillar: Agency Operations, AI Growth Systems, Client Delivery

## Useful Ideas

- AI work should live in a shared human and AI workspace, not only in chat windows.
- A task needs status, context, and an `AI ready` signal before an agent runs it.
- The human becomes the project manager and final QA gate.
- Low-friction capture matters because the bottleneck becomes scoping work accurately, not typing prompts.
- Evals are standardized checks that every AI output must pass before human review.

## Evidence

- 00:49 to 01:39: a shared AI and human workspace uses statuses such as inbox, next, doing, waiting, and done.
- 02:03 to 03:25: a task tagged `AI ready` is picked up by an agent, which consults the knowledge base, posts a plan, and starts work.
- 03:47 to 04:08: the human is the project manager, while the agent waits for approval before publishing or interacting with the wider internet.
- 05:45 to 06:09: agent outputs still require verification and human taste.
- 08:35 to 11:20: low-friction capture moves tasks from thought to the shared workspace, including desktop and phone capture.
- 12:23 to 15:10: evals are standardized checklists that test tone, reasoning, visual quality, expected value, verification, and leverage before the task reaches the human.
- 15:34 to 16:03: the work shifts from doing the deliverable to scoping and evaluating it.
