## Problem

A single metric can reward bad behavior. Speed can reduce quality. Volume can lower fit. Growth can hide unhappy customers.

## Framework

Use paired metrics that offset one another.

Examples:

- Speed + quality.
- Views + CPM or audience fit.
- Ascension / retention + CSAT or NPS.
- Delivery time + client approval quality.

## Workflow

For each BridgeWorks service, define one output metric and one quality guardrail:

- Website build: time to launch + checklist pass rate.
- Lead capture: number of leads + response time / lead quality.
- Content system: posts drafted + approved-post ratio.
- AI visibility: visibility score + accuracy of claims.
- Client delivery: tasks shipped + client acceptance notes.

Better reporting can make BridgeWorks look more serious than agencies that only show activity metrics.
