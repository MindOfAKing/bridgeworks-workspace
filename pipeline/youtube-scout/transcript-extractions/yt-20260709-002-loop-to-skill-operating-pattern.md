## Useful Ideas

- Move from one-off prompts to loops when the work needs repeated plan, build, test, verify cycles.
- A loop starts with an objective and ends with an exit condition.
- Good loops need clear scope, exact steps, and verification.
- Repeated loops can become skills.
- Examples fit BridgeWorks work: design review, SEO QA, conversion QA, content repurposing, and bug-fix verification.

## What This Changes For BridgeWorks

The durable rule is: if a task is repeated and has a testable end state, do not keep writing fresh prompts. Turn it into a loop card first, then a skill only after the loop works twice.

## Action To Take

Create a `Loop-To-Skill Gate` for BridgeWorks:

1. Is this repeated work?
2. What is the objective?
3. What are the loop steps?
4. What is the exit condition?
5. What proof confirms it worked?
6. What files or client data are allowed?
7. What approval gate stops external action?
8. Run twice before converting into a skill.
