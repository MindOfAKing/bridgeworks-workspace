## Useful Ideas

- Aggregate charts can hide whether individual users are getting value.
- A dot plot maps one user per row and one time period per column.
- Each dot should represent a value event, not vanity activity.
- For early products, a dot plot can be more useful than a full dashboard.
- B2B churn risk can be visible when bought seats do not activate or use the product repeatedly.

## Evidence

## What This Changes For BridgeWorks

BridgeWorks should not overbuild dashboards for Build Brief, DGE, or small client systems before the value event is clear.

For each product or client workflow, define one visible value event:

- Build Brief: useful brief generated and reviewed.
- DGE: insight accepted into an action.
- Foundation site: qualified enquiry submitted.
- Client reporting: owner-approved next action completed.
