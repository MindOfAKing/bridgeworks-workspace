## BridgeWorks fit

High fit for Build Brief, Foundation website production, and internal automation builds. The useful part is the sequence and handoff files, not the 24-hour promise.

BridgeWorks already uses AI as the build method. This source supports making every build start with: validated pain, AI-friendly PRD, roadmap with checkboxes, design.md, build loop, browser verification, and launch checklist.

## Adapted BridgeWorks rule

**Solo product sprint gate:** Before a new app, automation, or client-facing tool is built, require:

1. Clear buyer or internal user.
2. Pain signal or current workflow evidence.
3. AI-friendly PRD.
4. Roadmap with checkboxes.
5. `design.md` if a UI exists.
6. Build loop with code review.
7. Browser or workflow verification.
8. Launch checklist.
9. Manual approval before external publishing.

## Guardrails

- Do not promise 24-hour delivery publicly.
- Do not install new paid systems because the video promotes them.
- Do not launch Build Brief, client pages, or public assets without approval.
- Keep Foundation sites static-first unless scope and client need justify more.
