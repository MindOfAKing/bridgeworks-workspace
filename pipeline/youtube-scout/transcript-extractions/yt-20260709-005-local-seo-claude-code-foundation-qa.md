## Useful Ideas

- Local SEO starts with Google Business Profile structure before page production.
- GBP fields that matter most are categories, services, service areas, and reviews.
- Claude Code can draft GBP and page assets, but the owner must apply a truth filter.
- Service pages should map to money keywords.
- Local blog posts should target buyer-adjacent local searches.
- Static site generation and Lighthouse checks matter for technical SEO.
- Repeated page creation can become a reusable skill only after QA is clear.

## What This Changes For BridgeWorks

Add a `GBP-To-Site Content Map` before Foundation website page planning:

1. GBP primary category.
2. Valid secondary categories.
3. Real services.
4. Real service areas.
5. Photos and opening hours.
6. Review proof and response policy.
7. Money service pages.
8. Buyer-adjacent local content.
9. Static generation.
10. Lighthouse and on-page QA.

## Rejected Or Constrained

- No fake categories, fake service areas, or AI slop location pages.
- No review gating. Negative feedback can be handled internally, but customers must not be selectively blocked from leaving public reviews.
- No autoposting to GBP without client approval.
- No paid SEMrush, Make, domain, or tool spend from this run.
- No GitHub commit, push, or client publishing.
