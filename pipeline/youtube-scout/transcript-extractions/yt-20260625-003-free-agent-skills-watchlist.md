Medium-high. The tools fit three BridgeWorks needs: delivery verification, knowledge mapping, and content quality. But plugin bundles can create tool sprawl.

Best immediate fit:

1. GStack-style review pattern, not necessarily installation.
2. Stop Slop-style voice QA for Emmanuel's no-AI-slop rule.
3. Graphify / Understand Anything as a watch item for workspace maps.
4. Last 30 Days as a watch item for market sentiment research.

## Adapted BridgeWorks rule

**Skill adoption gate:** Before installing a skill or plugin into BridgeWorks work, record:

1. Exact use case.
2. Active project affected.
3. Free or paid status.
4. Data exposure risk.
5. Rollback path.
6. One proof task.
7. Keep / reject decision.

## Guardrails

- No plugin install in client repos from this cron job.
- No MCP connector expansion without approval.
- No paid adoption.
- No external publishing from generated content.
- Prefer extracting reusable behavior into local BridgeWorks checklists first.
